    Android 10 及更低版本

    提示

    使用 Android 11 或以上？ Android 11 指南更加方便。

    在 Android 10 和更早版本上，必须先在 USB 模式中启用 ADB。 然后你可以切换到网络或TCP/IP模式。 在 USB 模式中使用 ADB 需要计算机。

    将 Android 设备连接到 PC 并切换到 TCP/IP 模式是一个一次性过程。 但你需要在每次重启后重复。

    说明

    在继续操作之前，请确保您已启用 开发者选项。

    某些设备 (通常是自定义ROM) 在 Network 上有 ADB 或 无线调试 设置可在开发者选项中使用：

    在这种情况下，您不需要将设备连接到 PC。 只需在开发者选项中启用 ADB TCP/IP 模式。

    对于仅在 Develop Options 中使用 ADB USB 模式设置的其他设备，请遵循下面的说明。

    启用 TCP/IP¶

    • 在开发者选项中启用 Android 调试 / USB 调试 。

    • 使用 USB 线连接设备到电脑

    • 在 PC 上，打开终端窗口 (或在 Windows 上的命令提示) 然后运行：

      adb tcpip 5555

      其中 5555 是您希望在 TCP/IP 模式下启动 ADB 守护进程（adbd）的端口号。 您稍后将使用此端口号。

    提示

    您必须在计算机上安装 adb 可执行文件才能运行上述命令。 如果不是，这里是官方的 下载链接。

    一旦这些步骤完成，TCP/IP (无线调试) 应启用。

    验证 (可选)¶

    • 从您的设备上断开 USB 数据线连接

    • 将您的设备和电脑连接到同一个 Wi-Fi

    • 获取设备的IP地址（通常在Wi-Fi设置或设置 → 关于中），假设为 192.168.1.1。

    • 在 PC 上，运行：

      adb connect 192.168.1.1

    • 再次运行：

      adb shell id -u

    它应该打印 2000 （或 0）。

    将应用程序连接到ADB¶

    • 打开应用并点击ADB访问1。
    • 输入端口 5555 （来自上一节），然后点击连接。
    • 如需询问则允许 USB 调试 。

    说明

    您应该在开发者选项中保留启用的 USB 调试 。 禁用 ADB 将断开应用程序与 ADB 的连接。

    1. 如果ADB 访问 在抽屉里不可见， 按住应用程序图标/元件(在抽屉中)，除非显示ADB 访问复选框。 ↩
